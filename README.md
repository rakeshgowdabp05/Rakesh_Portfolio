## 📖 &nbsp;About This Project

**Rakesh Portfolio** is a personal developer portfolio website built by **Rakesh Gowda B P**, a Java Full Stack Developer (Fresher), to showcase his skills, education, projects, and experience to recruiters and collaborators.

This project is intentionally built as a **clean static frontend** — no frameworks, no build tools, no backend server, and no database. Every animation, transition, and interaction is hand-written in **vanilla HTML, CSS, and JavaScript**, which reflects a solid grasp of core web fundamentals without relying on abstractions.

> 💡 **Why static & framework-free?** As a fresher applying for full-stack roles, this project demonstrates that the fundamentals — semantic HTML, responsive CSS, and DOM-driven JS — are solid before ever reaching for a framework.

## 🎨 &nbsp;Design Philosophy

The portfolio follows a **dark, premium aesthetic** tailored for a Java Full Stack Developer fresher:

- 🌑 **Dark theme base** with a professional **green/teal accent** (`#2DD4BF` / `#14b8a6` family) for a calm, modern, tech-forward feel
- 🃏 **Aligned, consistent cards** for skills, projects, and experience with matching padding and spacing
- ✍️ **Readable typography** with clear hierarchy between headings, subheadings, and body text
- 🌊 **Smooth transitions** on hover, scroll-reveal, and section changes — nothing jarring, everything eased
- 📐 **Responsive grid layout** that reflows gracefully from desktop → tablet → mobile
- 🧘 **Generous whitespace** so the page never feels cluttered, keeping focus on content

## 📬 &nbsp;Contact Form (Web3Forms)

The contact section uses **[Web3Forms](https://web3forms.com/)** to deliver messages directly to Gmail — no custom backend, no database, no server code required.

**Form fields:**

- 👤 Name
- 📧 Email
- 📝 Subject
- 💬 Message
- 🕵️ Hidden `botcheck` field (spam protection / honeypot)

**Submission flow:**

```
User fills form → Submits → Web3Forms delivers to Gmail inbox
        → Success message shown → Redirects to thank-you.html
```

No API keys are exposed beyond the public Web3Forms access key, which is safe to expose client-side by design.

## ⚙️ &nbsp;Running Locally

Since this is a **pure static website**, no build step or package installation is required.

### Option 1 — Just open it
Double-click `index.html` and it opens directly in your browser.

### Option 2 — Serve it with Java's built-in web server
As a Java developer, you can spin up a quick local server using Java's native `jwebserver`:

```bash
jwebserver -p 5500
```

Then visit:

```
http://localhost:5500
```

### Option 3 — VS Code Live Server
If you use VS Code, right-click `index.html` → **Open with Live Server**.

## 🚀 &nbsp;Deployment

This project is deployed on **Vercel** with zero configuration — since there's no build step, Vercel serves the static files directly.

**Live URL:** [rakesh05-portfolio.vercel.app](https://rakesh05-portfolio.vercel.app)

To deploy your own copy:

1. Fork or clone this repository
2. Go to [vercel.com](https://vercel.com) → **New Project**
3. Import the GitHub repo
4. Leave build settings empty (static site)
5. Deploy 🎉
